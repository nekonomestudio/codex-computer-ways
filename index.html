Codex 使用電腦的三種方式

由 OpenAI 開發團隊工程師提出的 Codex 電腦操作指南：用 Computer Use、Chrome extension 與 in-app Browser 三種介面，對應不同權限、上下文與工作場景。

三種方式怎麼選

判斷重點不是「哪個最強」，而是哪個介面剛好提供任務需要的權限與上下文。

工具 最適合 主要特點 觸發方式
@Computer 桌面 App、系統設定、iOS 模擬器、iPhone Mirroring、沒有 API 的流程。 範圍最廣，能看畫面、點選、打字、跨 App 操作；通常也最慢。 提到 @Computer，或明確要求使用 Computer Use。
@Chrome 需要你已登入 Chrome 的任務，例如 Gmail、LinkedIn、Salesforce、內部後台。 能使用瀏覽器身分、cookies、既有分頁與多分頁上下文。 提到 @Chrome，或要求使用已登入的 Chrome。
@Browser 本機 web app、公開頁面、RWD 檢查、視覺 bug 重現、設計標註。 與 Codex 共享同一個隔離瀏覽器畫面；適合開發與除錯。 提到 @Browser，或要求使用 in-app browser。
優先原則：能用 Plugin、MCP 或 API 時，先用結構化工具。視覺控制最適合用在「結構化工具到不了的最後一段」。
01

Computer Use

最廣泛的操作介面。Codex 可以透過視窗、選單、鍵盤、滑鼠與剪貼簿，操作你批准的桌面應用程式。

特點

它能處理沒有 API 或 plugin 的圖形介面，也能跨 App 串起工作流。代價是需要看畫面、判斷、點擊、等待回應，所以速度通常比結構化工具慢。

適合用在

  • Spotify、Xcode、財務 App 等原生桌面軟體
  • 系統設定、應用程式設定
  • iOS 模擬器、iPhone Mirroring
  • 沒有 API 的資料來源或流程
  • 結構化整合缺少最後一步操作時

例子

作者曾讓 Codex 定時查看 Amazon 客服聊天，客服出現後協助處理退款；也曾在 Slack plugin 不能上傳影片時，用 Computer Use 點擊「Add file」完成最後一步。

桌面 App 操作 Trigger：提到 @Computer 
Use @Computer to open Spotify, find my Discover Weekly playlist, and start it. Do not change my account or subscription settings.
iOS 流程重現 Trigger：要求使用 Computer Use 
Use @Computer to open iPhone Mirroring, reproduce the onboarding bug in the iOS app, and take a screenshot of the failing state. Fix the smallest relevant code path, then run the same flow again.

需要注意

這是三者中最大的信任邊界。一次只給它清楚的一個 App 或流程；敏感 App 不用時保持關閉；涉及付款、帳戶、憑證、隱私與系統安全時，要保留人工確認。

02

Chrome Extension

讓 Codex 使用你已登入的 Chrome 狀態。當任務需要帳號、cookies、瀏覽器 profile 或既有分頁時，就該考慮它。

特點

Chrome 能把工作理解成瀏覽器流程，而不是螢幕座標。它可以保留同一任務的多個分頁，讀 A、比較 B、繼續操作 C。

適合用在

  • Gmail、LinkedIn、X 等登入後服務
  • Salesforce、客服 console、內部 dashboard
  • 需要跨多個網站查資料的研究
  • 依賴帳號、cookies 或瀏覽器 extension 的表單

例子

作者描述了一個長期 thread：每天用 Chrome 檢查 DM、讀相關新聞、找重要 mention，把有長期價值的內容寫入 vault，但明確要求不要發文或傳訊息。

多分頁帳戶資料整理 Trigger：提到 @Chrome 
Use @Chrome to review the open customer account, compare it with the support ticket in the other tab, and draft the missing fields. Stop before submitting.
長期登入狀態研究 Trigger：要求使用已登入的 Chrome 
Every day, use Chrome to check my DMs, read relevant news, and look for feedback or mentions I should know about. Add anything durable to my vault. Do not post or send messages.

需要注意

網站會把 Codex 的點擊、送出表單和訊息視為你本人操作。適合自動研究、瀏覽、草稿；送出、發布、購買或提交前，應該要求人工審核。

如果整個任務都在瀏覽器裡，通常應該優先用 Chrome，而不是讓 Computer Use 以視覺方式操作瀏覽器。
03

In-app Browser

活在 Codex thread 裡的隔離瀏覽器。你和 Codex 看到同一個渲染頁面，所以特別適合一起做網站開發、視覺檢查與設計修正。

特點

Codex 可以改 code、操作頁面、檢查渲染結果、截圖，再回頭修正。這個迴圈讓 web app 除錯和 UI 迭代變得很短。

適合用在

  • 本機開發伺服器
  • file-backed previews
  • 不需要登入的公開頁面
  • 重現視覺 bug、檢查 responsive layout
  • 對具體元素留下設計回饋

例子

作者常讓 Codex 先做一個單檔 index.html，再用 in-app Browser 打開。他會直接在頁面上標註：「層級反了」、「不要那麼像卡片」、「這些控制項需要更多空間」，Codex 收到截圖與元素上下文後再修改。

本機網頁除錯 Trigger：提到 @Browser 
Use @Browser to open vite app on http://localhost:3000/, reproduce the mobile overflow bug, fix it, and verify the same route again at desktop and mobile widths.
單頁設計迭代 Trigger：要求使用 in-app Browser 
Create a single-file index.html for this project brief and open it in the in-app @Browser.

需要注意

它不使用你的日常瀏覽器 profile、cookies、extension、登入狀態或現有分頁。這是很好的安全邊界，但不適合處理 Google login、passkey 或依賴帳號的網站。

Appshots 的位置

Appshots 不是第四種控制電腦的方法，而是把你眼前的畫面指給 Codex 看。在 macOS 的 Codex app 中，可以用 Cmd + Cmd 擷取最前方視窗，附上圖片與可取得文字。

簡單記法：Appshots 用來「指出上下文」；Browser、Chrome、Computer Use 則用來「採取行動」。
